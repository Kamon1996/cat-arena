Минимальная продакшен-схема:

Подход №1. Placement matches

Самый популярный вариант.

Новому коту дается:

Например:

или

первые 50 голосов

Для новых котов действует буст.

Например:

В результате новый кот быстро набирает стартовый рейтинг.

Как подобрать пару

Я бы делал так.

Допустим нужен кот:

ELO = 1500

Берем окно:

±100 рейтинга

Получаем кандидатов:

1400 - 1600

Из них считаем вес:

weight =
freshnessBoost *
exposureBoost *
randomness

Где:

freshnessBoost

зависит от возраста кота.

exposureBoost

от количества показов.

randomness

дает небольшую случайность.

Что бы я сделал в твоем проекте

Для MVP:

1. ELO рейтинг.

2. Подбор только в диапазоне ±100 ELO.

3. Новые коты получают буст до первых 20 голосов.

4. Учитываем количество показов:
   меньше показов → выше шанс попасть в выборку.

5. При генерации очереди из 5 пар
   один и тот же кот не может попасть дважды.

6. Пользователю не показывать котов,
   которых он видел последние 100 показов.

Такой алгоритм уже будет выглядеть очень "живым": новые коты быстро получают аудиторию, старые не монополизируют показы, а качество рейтинга остается достаточно высоким.
