# Пошаговый план переезда на продакшен (личный walkthrough)

> Цель: перенести cat-arena с бесплатных тиров (Vercel + Neon + R2 + Upstash) на **всё на Timeweb
> Cloud** (сервер + managed PostgreSQL + S3-хранилище + CDN) + домен **.com на reg.ru**. Одна панель,
> один счёт. Инструкция «зайди туда → нажми то → зачем», по фазам. Прод на Vercel **не трогаем**,
> пока новый стек не готов.
>
> Код уже подготовлен (Docker/standalone, миграции-сервис, S3-через-env, переключаемый Redis,
> cron, health, graceful shutdown). Поэтому переезд = «поднять сервисы + заполнить `.env`».
>
> ⚠️ Цены — проверено в июне 2026, **сверяй в калькуляторе провайдера перед оплатой** (с
> 01.01.2026 НДС вырос до 22%, тарифы могли подрасти).

## Фаза 0. Подготовка (бесплатно)

1. Аккаунты: **timeweb.cloud** (всё: сервер + БД + S3 + CDN) и **reg.ru** (домен). Оба — рублёвая
   оплата картой Мир/РФ-Visa/MC или СБП, без зарубежных карт.
2. Пополни баланс Timeweb (**списывает почасово и приостанавливает сервисы при нуле** — держи плюс).
3. SSH-ключ: `ssh-keygen -t ed25519 -C cat-arena-vps` (если нет). Публичный — `~/.ssh/id_ed25519.pub`.

## Фаза 1. Домен .com на reg.ru (~15 мин, 1792 ₽/год)

1. **reg.ru → «Войти/Регистрация»** (email + телефон).
2. На [reg.ru/domain/new](https://www.reg.ru/domain/new/) введи имя → выбери строку **.com**
   («В корзину»). Цена: **.com = 1792 ₽ первый год** (промо; стандарт 2819 ₽). ⚠️ продление ~2700–2900 ₽/год.
3. В оформлении укажи контакты (физлицо ок). ⚠️ **WHOIS-приватность для .com — НЕ бесплатна
   автоматически** (бесплатна только для .ru/.su/.рф). Хочешь скрыть данные — это платная опция,
   проверь в корзине. Откажись от ненужных допов (SSL — у нас Caddy бесплатно; хостинг).
4. **Оплати** картой/СБП. .com регистрируется за минуты.
5. **Неймсерверы — оставь reg.ru** (`ns1.reg.ru` / `ns2.reg.ru`). ⚠️ **Не подключай Cloudflare-прокси**
   — он троттлится в РФ. DNS-записи добавим в Фазах 4–5, когда будут IP/CDN-хост.

> Документы: DNS-зона редактируется в `Личный кабинет → Домены → клик по домену →
> «DNS-серверы и управление зоной» → «Изменить» → «Добавить запись»`.

## Фаза 2. Сервер на Timeweb Cloud (~30 мин, ~1062 ₽ + 180 ₽ IPv4)

1. **timeweb.cloud → «Облачные серверы» → вкладка «SSH-ключи» → «Добавить»** — вставь публичный
   ключ, имя `cat-arena`, галка «Автоматически добавлять на новые серверы». *(Сделай ДО создания
   сервера — тогда можно выбрать ключ и отключить вход по root-паролю.)*
   [Док](https://timeweb.cloud/docs/cloud-servers/manage-servers/ssh-keys)
2. **«Облачные серверы» → «Создать»**. Шаги мастера (актуальные названия):
   - **Программное обеспечение** → Ubuntu → **22.04 LTS**.
   - **Регион и зона доступности** → **Москва** (линейка Cloud MSK). ⚠️ Запомни регион — БД в Фазе 3
     должна быть в **том же регионе** для приватной сети.
   - **Конфигурация** → вкладка Premium 3.3 ГГц → **Cloud MSK 50** (2 vCPU / 4 ГБ / 50 ГБ NVMe).
   - **Сеть** → ⚠️ **добавь публичный IPv4** («Создать новый», +180 ₽/мес) — он НЕ включён по
     умолчанию, без него Caddy не получит TLS. (Опционально создай и приватную сеть — для БД.)
   - **Безопасность** → можно включить бэкапы/DDoS.
   - **Авторизация** → выбери свой SSH-ключ и **выключи «Доступ по SSH через root-пароль»**.
   - **Информация о сервере** → задай имя хоста. → «Заказать».
   [Док](https://timeweb.cloud/docs/cloud-servers/manage-servers/create-server)
3. Через ~1 мин статус «Создан». Открой сервер → вкладка **«Дашборд»**: там публичный IPv4 и готовая
   команда `ssh root@<IP>`. Запиши IP.
4. **DNS:** вернись в reg.ru (Фаза 1) → DNS-зона → добавь:
   - запись **A**, поддомен `@`, значение = IP сервера;
   - запись **A**, поддомен `www`, значение = тот же IP.

### 2.1. Базовая настройка сервера (по SSH)
```bash
ssh root@<IP>
apt update && apt -y upgrade
# отдельный пользователь
adduser deploy && usermod -aG sudo deploy
rsync --archive --chown=deploy:deploy ~/.ssh /home/deploy
# фаервол: только 22, 80, 443 (НЕ открывай 3000 — Caddy проксирует внутри; Redis в compose-сети)
ufw allow OpenSSH && ufw allow 80 && ufw allow 443 && ufw enable
# Docker + compose
curl -fsSL https://get.docker.com | sh
usermod -aG docker deploy
docker --version && docker compose version
```

## Фаза 3. Managed PostgreSQL на Timeweb (~15 мин, ~790 ₽)

1. **timeweb.cloud → «Базы данных» → «Добавить»** → движок **PostgreSQL**.
2. Версия: **PostgreSQL 16** ⚠️ (версия **НЕ меняется** после создания; 16 совпадает с
   `postgres:16` в нашем docker-compose).
3. Конфигурация: **Cloud DB 1/2/20** (1 CPU / 2 ГБ / 20 ГБ) ~790 ₽ — для лёгкой нагрузки хватает.
   *(Cloud DB 1/1/8 ~230 ₽ — самый дешёвый, но 1 ГБ RAM тесновато.)*
4. Регион: **тот же, что у сервера** (Москва) — для приватной сети.
5. **Публичный IPv4 для БД — НЕ включай** (экономит +180 ₽): подключим по приватной сети.
6. Включи **физические бэкапы**.
   [Док](https://timeweb.cloud/docs/dbaas/dbaas-create)
7. **Доступ к БД** (вкладка «Сеть» кластера): подключи БД к **той же приватной сети**, что и сервер,
   и **отключи публичный IPv4 БД** («Разрешить доступ по публичному IPv4-адресу» → выкл). Тогда БД
   доступна только из приватной сети. *(TLS включён по умолчанию для БД, созданных после 01.07.2025.)*
8. **Строка подключения** (вкладка «Дашборд»/«Подключение»): панель соберёт готовый
   `postgresql://user:pass@host:5432/db`. ⚠️ **У Timeweb DBaaS нет отдельного пулера (PgBouncer)** —
   `DATABASE_URL` и `DIRECT_URL` указывают на **один и тот же** host:5432. Поэтому:
   - `DIRECT_URL = postgresql://USER:PASS@HOST:5432/DB?sslmode=require`
   - `DATABASE_URL = postgresql://USER:PASS@HOST:5432/DB?sslmode=require&connection_limit=10&pool_timeout=20`
   - ⚠️ **НЕ добавляй `pgbouncer=true`** (это был артефакт Neon — пулера за хостом нет, флаг зря
     отключит prepared statements). Одна реплика app = один пул, `connection_limit=10` достаточно.

### 3.1. Перелив данных из Neon
```bash
# дамп из Neon (DIRECT_URL Neon из текущего .env.local)
pg_dump "$NEON_DIRECT_URL" -Fc -f catarena.dump
# restore в Timeweb (по публичному IP временно, либо с сервера по приватному)
pg_restore -d "postgresql://USER:PASS@HOST:5432/DB?sslmode=require" --no-owner --no-privileges catarena.dump
```
*(Миграции Prisma накатит сам compose-сервис `migrate` при первом `docker compose up`.)*

## Фаза 4. Хранилище + CDN на Timeweb (~30 мин, ~349 ₽ + трафик)

### 4.1. S3-бакет
1. **timeweb.cloud → «Облачное хранилище S3» → «Создать»**: имя бакета (напр. `cat-arena-images`),
   тип доступа — **Публичный** (чтение объектов), регион **ru-1**. Тариф 100 ГБ (~349 ₽) или
   per-use. [Услуга](https://timeweb.cloud/services/s3-storage) · [Док](https://timeweb.cloud/docs/s3-storage)
2. На странице созданного бакета (раздел подключения) возьми **Access Key** и **Secret Key**.
   Параметры подключения фиксированы: endpoint `https://s3.twcstorage.ru`, регион `ru-1`.
   *(Ключи привязаны к проекту/бакету — берутся прямо в панели, не нужно создавать сервисные аккаунты.)*

### 4.2. CORS (нужен для прямой загрузки из браузера по presigned-URL)
3. Бакет → **«Настройка CORS»**: AllowedOrigins = `https://твойдомен.com`,
   AllowedMethods = `GET, PUT`, AllowedHeaders = `*`, MaxAgeSeconds = `3600`. *(Серверные загрузки
   CORS не требуют, но у нас браузер PUT-ит напрямую → правило обязательно.)*

### 4.3. Перелив картинок из R2 (rclone S3→S3)
```bash
# ~/.config/rclone/rclone.conf
# [r2]  type=s3 provider=Cloudflare access_key_id=... secret_access_key=... endpoint=https://<acct>.r2.cloudflarestorage.com
# [tw]  type=s3 provider=Other  access_key_id=<AccessKey> secret_access_key=<Secret> endpoint=https://s3.twcstorage.ru region=ru-1
rclone copy r2:cat-arena tw:cat-arena-images --transfers 16 --checkers 16 --progress
```
⚠️ provider = **Other**. Ключи объектов сохраняются → в БД переписывать ничего не нужно.

### 4.4. CDN-ресурс (origin = S3-бакет, тот же провайдер)
4. **timeweb.cloud → «CDN» → «Добавить»**: «Источник контента» = **«S3-бакет»** → выбери
   `cat-arena-images`. Задай имя ресурса → «Заказать». [Док](https://timeweb.cloud/docs/cdn/create)
5. Ресурс получит **технический CDN-домен** (виден на дашборде ресурса) — это таргет для CNAME.
6. **«Настройки ресурса» → «Источник контента и домены раздачи»**: добавь свой домен
   `cdn.твойдомен.com`.
7. **SSL**: в разделе SSL ресурса включи выпуск **Let's Encrypt** (Timeweb выпустит сам после того,
   как заработает CNAME из шага 8). [Док](https://timeweb.cloud/docs/cdn/ssl)
8. **DNS в reg.ru**: добавь запись **CNAME**, поддомен `cdn`, значение = технический CDN-домен из
   шага 5. ⚠️ `cdn` — только CNAME (нельзя A+CNAME на одном имени); apex `@` остаётся A на сервер.

> 🔁 **CDN не нравится по скорости в каких-то регионах?** Переключение на Yandex/Gcore CDN — это
> смена двух env (`S3_ENDPOINT` + `R2_PUBLIC_URL`) и одного CNAME, без правок кода.

## Фаза 5. Деплой приложения (~30 мин)

На сервере под `deploy`:
```bash
git clone <repo-url> cat-arena && cd cat-arena
cp .env.example .env
nano .env   # заполни (см. таблицу ниже)
docker compose up -d --build
```
Compose поднимет: `migrate` (накатит `prisma migrate deploy` и выйдет) → `app` (Next.js standalone)
+ `redis` + `caddy` (авто-TLS). Проверь: `https://твойдомен.com/api/health` → `{"status":"ok"}`.

### `.env` — что куда (финальный маппинг)
| Переменная | Значение |
|---|---|
| `DATABASE_URL` | `postgresql://USER:PASS@HOST:5432/DB?sslmode=require&connection_limit=10&pool_timeout=20` (Timeweb PG) |
| `DIRECT_URL` | `postgresql://USER:PASS@HOST:5432/DB?sslmode=require` (Timeweb PG, тот же host) |
| `REDIS_DRIVER` | `redis` |
| `REDIS_URL` | `redis://redis:6379` (контейнер из compose) |
| `S3_ENDPOINT` | `https://s3.twcstorage.ru` (Timeweb S3) |
| `S3_REGION` | `ru-1` |
| `S3_FORCE_PATH_STYLE` | `true` |
| `R2_ACCESS_KEY_ID` | Access Key со страницы бакета Timeweb |
| `R2_SECRET_ACCESS_KEY` | Secret Key со страницы бакета Timeweb |
| `R2_BUCKET` | `cat-arena-images` |
| `R2_PUBLIC_URL` | `https://cdn.твойдомен.com` ⚠️ (хост Timeweb CDN, НЕ s3.twcstorage.ru — иначе платный S3-egress) |
| `DOMAIN` | `твойдомен.com` |
| `AUTH_URL` | `https://твойдомен.com` ⚠️ (полный origin, БЕЗ слэша в конце) |
| `AUTH_SECRET`, `PAIR_TOKEN_SECRET`, `CRON_SECRET` | сгенерируй: `openssl rand -base64 32` |
| `AUTH_GOOGLE_ID` / `AUTH_GOOGLE_SECRET` | из Google Cloud Console |
| `CLOUDFLARE_ACCOUNT_ID` / `CLOUDFLARE_API_TOKEN` | для AI-модерации (Workers AI) — остаётся как есть |
| `NEXT_PUBLIC_POSTHOG_KEY` / `NEXT_PUBLIC_POSTHOG_HOST` | PostHog (или замени на Метрику позже) |
| `SENTRY_DSN` | Sentry или self-host GlitchTip |

⚠️ **Google OAuth:** в Google Cloud Console → APIs & Services → Credentials → твой OAuth client →
Authorized redirect URIs добавь `https://твойдомен.com/api/auth/callback/google`.

## Фаза 6. Cron (RD-decay + чистка сирот)

Код готов (`/api/cron/decay`, `/api/cron/cleanup`, защита `CRON_SECRET`). На сервере — системный crontab:
```bash
crontab -e
# RD-decay — раз в сутки
0 3 * * * curl -fsS -H "Authorization: Bearer ВАШ_CRON_SECRET" https://твойдомен.com/api/cron/decay
# чистка R2-сирот — раз в неделю
0 4 * * 0 curl -fsS -H "Authorization: Bearer ВАШ_CRON_SECRET" https://твойдомен.com/api/cron/cleanup
```

## Жёсткие напоминания

- 🔴 **Публичный IPv4 у Timeweb платный** (+180 ₽/мес) — добавь на шаге «Сеть», иначе нет TLS.
- 🔴 **S3-ключи (Access/Secret)** бери на странице бакета Timeweb; endpoint `https://s3.twcstorage.ru`, регион `ru-1`.
- 🔴 **`R2_PUBLIC_URL` = хост Timeweb CDN**, не `s3.twcstorage.ru` (иначе платишь за прямой S3-egress).
- 🔴 **Не добавляй `pgbouncer=true`** в Timeweb (пулера нет) — используй `connection_limit=10`.
- 🔴 **Порты 80 и 443 открыты** на сервере — иначе Caddy не выпустит Let's Encrypt.
- 🔴 **Без Cloudflare-прокси** — троттлится в РФ. DNS у reg.ru, CNAME на Timeweb CDN.
- 🟡 БД и сервер — **в одном регионе** (приватная сеть, низкая задержка).
- 🟡 `.com` — промо 1792 ₽ только 1-й год; включи автопродление (продление ~2700–2900 ₽/год).
- 🔁 CDN можно позже сменить на **Yandex/Gcore** (сменой `S3_ENDPOINT` + `R2_PUBLIC_URL`), если
  понадобится более широкое глобальное покрытие — кода это не касается.
