# Scaffold & Tooling Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox syntax for tracking.

**Goal:** Bootstrap the cat-arena Next.js 15 app into the existing repo with a complete, passing tooling baseline (Tailwind v4, CVA, shadcn/ui, Biome, Stylelint, git hooks, Vitest + RTL, Playwright, a Zod-validated env module, strict TypeScript) so `npm run build` and every check pass.

**Architecture:** A single Next.js 15 App Router project rooted at the repo (TypeScript, `src/` dir, `@/*` import alias) where Biome owns JS/TS/JSON lint+format, Stylelint owns CSS correctness, lefthook runs `biome check` + `stylelint` + `tsc --noEmit` on pre-commit, and tests split into Vitest/RTL unit+component tests (jsdom) and Playwright browser E2E. All external config (env vars) is validated once through `src/lib/env.ts` with Zod 4 and the canonical names live in `src/lib/constants.ts`.

**Tech Stack:** Next.js 15 (App Router, RSC) · React 19 · TypeScript 5 (strict) · Tailwind CSS v4 · class-variance-authority · shadcn/ui + lucide-react + sonner · Biome v2 · Stylelint + stylelint-config-standard · lefthook · Vitest 4 + @testing-library/react · Playwright · Zod 4.

- **Named exports only**, except Next.js `page`/`layout`/`route`/`sitemap`/`robots` (and shadcn-generated `ui/*` which export named primitives). Biome's `noDefaultExport` override (Task 3) enforces this split.
- **`verbatimModuleSyntax`** is on — every type-only import uses `import type`. The provided code already complies (e.g. none of the lib/test files import a type as a value).
- **No magic numbers:** test files and configs use named constants (`MIN_SECRET_LENGTH`, `PORT`, `EXPECTED_TITLE`, etc.) per CLAUDE.md and the testing skills.
- **`env.ts` import-time validation:** the test imports only `envSchema`, never the `env` constant, so running tests does not require a populated environment.
- **Dependency provenance:** `class-variance-authority`, `clsx`, `tailwind-merge` arrive via `shadcn add`; `zod` is added explicitly (Task 6) because both `env.ts` and all later API/form contracts depend on it.
