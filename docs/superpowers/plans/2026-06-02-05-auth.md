# Auth (magic-link) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox syntax for tracking.

**Goal:** Add passwordless magic-link authentication (Auth.js v5 + Prisma adapter, DB sessions) that sends sign-in links via Resend/React Email, exposes a `/signin` page, gates `/upload`, `/orgs/new`, and `/admin` behind a session while leaving voting anonymous, and lets `/api/vote` attribute votes to a logged-in user.

**Architecture:** Auth.js v5 is configured in `src/auth/config.ts` (full config with `PrismaAdapter` + `session.strategy = "database"` + a custom Resend magic-link provider) and re-exported from `src/auth/index.ts` as `{ handlers, auth, signIn, signOut }`. A slim, Prisma-free `src/auth/edge.ts` powers `src/middleware.ts` so route protection runs on the Edge runtime (Next.js 15) without importing the database client; per-route server guards in `src/auth/guards.ts` provide defense-in-depth. The magic-link email is a typed React Email component in `src/auth/email.tsx`, rendered and sent through the Resend SDK inside the provider's `sendVerificationRequest`.

**Tech Stack:** Auth.js v5 (`next-auth@5`), `@auth/prisma-adapter`, `resend` SDK, `@react-email/components` + `@react-email/render`, Next.js 15 App Router (middleware + Server Components), Zod (env), Vitest + Playwright (tests).

- [ ] **Step 2: Write `src/auth/send-magic-link.ts` with FULL code**
  - Auth.js calls `sendVerificationRequest({ identifier, url, provider })`. We render our template and send via Resend, throwing on error so Auth.js routes the user to the error page rather than silently "succeeding". Full code:
    ```ts
    import { Resend } from "resend";

    import { AUTH } from "@/lib/constants";
    import { env } from "@/lib/env";

    import { renderMagicLinkEmail } from "./email";

    const resend = new Resend(env.RESEND_API_KEY);

    interface SendVerificationRequestParams {
      identifier: string;
      url: string;
    }

    export async function sendMagicLink({
      identifier,
      url,
    }: SendVerificationRequestParams): Promise<void> {
      const html = await renderMagicLinkEmail(url);

      const { error } = await resend.emails.send({
        from: env.EMAIL_FROM,
        to: identifier,
        subject: AUTH.EMAIL_SUBJECT,
        html,
      });

      if (error) {
        throw new Error(`Failed to send magic-link email: ${error.message}`);
      }
    }
    ```
  - Note: `env` is the validated object exported from `src/lib/env.ts`. If `env.ts` exposes `process.env` access under a different name, adjust the import to match its public API (do not read `process.env` directly here).

    ```ts
    import { AUTH } from "@/lib/constants";

    export const edgeAuthConfig = {
      providers: [],
      pages: {
        signIn: AUTH.SIGN_IN_PATH,
      },
      callbacks: {
        authorized({ auth, request }) {
          const { pathname } = request.nextUrl;
          const isProtected = AUTH.PROTECTED_PREFIXES.some((prefix) =>
            pathname.startsWith(prefix),
          );
          if (!isProtected) return true;

          const isLoggedIn = Boolean(auth?.user);
          if (!isLoggedIn) return false; // false -> redirect to pages.signIn

          if (pathname.startsWith(AUTH.ADMIN_PREFIX)) {
            return auth?.user?.role === "ADMIN" || auth?.user?.role === "MODERATOR";
          }
          return true;
        },
      },
    } satisfies NextAuthConfig;
    ```

    ```ts
    import { AUTH } from "@/lib/constants";

    import { auth } from "./config";

    import type { Session } from "next-auth";

    export async function requireUser(): Promise<Session> {
      const session = await auth();
      if (!session?.user) {
        redirect(AUTH.SIGN_IN_PATH);
      }
      return session;
    }

    export async function requireModerator(): Promise<Session> {
      const session = await requireUser();
      if (session.user.role !== "MODERATOR" && session.user.role !== "ADMIN") {
        redirect("/");
      }
      return session;
    }

    export async function requireAdmin(): Promise<Session> {
      const session = await requireUser();
      if (session.user.role !== "ADMIN") {
        redirect("/");
      }
      return session;
    }
    ```

    ```ts
    import { cookies } from "next/headers";

    import { auth } from "@/auth";
    import { AUTH } from "@/lib/constants";

    import type { Session } from "next-auth";

    interface ResolveVoterKeyDeps {
      getSession: () => Promise<Session | null>;
      getAnonId: () => Promise<string>;
    }

    export async function resolveVoterKey({
      getSession,
      getAnonId,
    }: ResolveVoterKeyDeps): Promise<string> {
      const session = await getSession();
      if (session?.user?.id) {
        return `user:${session.user.id}`;
      }
      return getAnonId();
    }

    async function readAnonId(): Promise<string> {
      const store = await cookies();
      return store.get(AUTH.ANON_COOKIE)?.value ?? "";
    }

    export async function getVoterKey(): Promise<string> {
      return resolveVoterKey({
        getSession: () => auth(),
        getAnonId: readAnonId,
      });
    }
    ```
