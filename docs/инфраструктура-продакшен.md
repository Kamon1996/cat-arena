## ★ ТВОЙ ВЫБОР: российский VPS-стек (готов к DevOps, аудитория международная)

**Логика:** российский VPS не троттлится для РФ-юзеров И доступен из-за рубежа (троттлинг —
только РФ-ISP → Cloudflare, не обратно). Один стек покрывает обе аудитории. Оплата зарубежных
карт — отложена (не нужна на российском стеке).

**Рекомендованный конкретный стек — Selectel (золотая середина для VPS-пути):**

| Компонент | Решение | ~Цена/мес ⚠️ |
|---|---|---|
| Compute + Redis + cron | **1 VPS Selectel, 4 vCPU / 8 GB**: Docker Compose → `next start` (standalone) + **Caddy** (авто-TLS Let's Encrypt) + Redis-контейнер + crontab | ~1500–2500 ₽ |
| База | **Selectel Managed PostgreSQL** — ⭐ единственный РФ-managed со **встроенным PgBouncer** (чисто ложится на Prisma), бэкапы/PITR из коробки | ~1500–2500 ₽ |
| Storage | **Selectel Object Storage** (budget-класс 0.79 ₽/ГБ) | ~200–400 ₽ |
| CDN | **Selectel CDN** (55+ PoP РФ/СНГ, перед bucket) | по трафику ~200–900 ₽ |
| Домен | **.ru** на reg.ru (169 ₽ 1-й год) + опц. .com для бренда | ~15–100 ₽/мес |
| Email | **Sendersy** (free 3000/мес) | 0 ₽ старт |
| Ошибки | **GlitchTip self-host** на том же VPS (256 МБ–1 ГБ, тот же `@sentry/nextjs` SDK) | 0 ₽ |
| Аналитика | **Яндекс.Метрика** (free, РФ-хостинг) или PostHog cloud (free 1M) | 0 ₽ |
| Uptime | **Betterstack** free / **Gatus** self-host (⚠️ НЕ UptimeRobot — запрещён commercial) | 0 ₽ |
| **ИТОГО** | | **~3500–6500 ₽/мес** |

**Почему Selectel, а не Timeweb/Yandex для тебя:**
- vs **Timeweb** (дешевле, ~1200–2500 ₽): у Timeweb managed-PG беднее и **без встроенного пулера** —
  тебе пришлось бы поднимать PgBouncer руками. Selectel этот пулер даёт managed.
- vs **Yandex Cloud** (дороже, ~5000–9000 ₽): Yandex — максимум managed, но ты «готов к VPS», так
  что переплата за их managed-VM/Valkey не нужна. Плюс у Yandex пулер — Odyssey (сложнее
  `DIRECT_URL`), у Selectel — нормальный PgBouncer.

**Бюджетный вариант (максимум экономии):** Timeweb VPS (~550 ₽) + self-host Postgres на том же
VPS (pg_dump→S3 еженедельно + PgBouncer-контейнер) + Timeweb S3/CDN. ~1200–2000 ₽/мес. Но Postgres
полностью на тебе (бэкапы, failover, пулер) — это **реальный ops-риск**, выстрелит в худший момент.

**Что переписать в коде (главное):**
1. ⚠️ **rate-limit** — `@upstash/ratelimit` не работает с обычным Redis. Переписать на `ioredis` +
   Lua или `rate-limiter-flexible`. **Самая трудоёмкая часть миграции.**
2. `S3_ENDPOINT` + credentials → Selectel (S3-код не трогаем).
3. `DATABASE_URL`/`DIRECT_URL` → Selectel PG (PgBouncer-эндпоинт, `?pgbouncer=true`).
4. `output: 'standalone'` в `next.config.ts` (для Docker-образа).
5. Resend → Sendersy (проверить рендер React Email → HTML).
6. Добавить cron: RD-decay (раз/сутки) + чистка R2-сирот (раз/неделю).

Пошаговый план миграции — см. §7 ниже.

## 0. Главная развилка: КТО твои пользователи?

Это решает почти всё, из-за двух жёстких ограничений, появившихся в 2025:

1. **Троттлинг Cloudflare в РФ (с 9 июня 2025).** Российские ISP (Ростелеком, МТС, Мегафон,
   Билайн) режут трафик через IP Cloudflare до **16 КБ на соединение** по директиве РКН.
   Картинки с R2/Cloudflare CDN у российских пользователей просто **не догрузятся**. Тот же
   троттлинг затрагивает Hetzner, DigitalOcean, OVH. → Для РФ-аудитории зарубежный хостинг и
   Cloudflare-CDN **нежизнеспособны**.
2. **152-ФЗ, ужесточён с 1 июля 2025.** Персональные данные граждан РФ (а у тебя это email +
   Google OAuth) должны **первично собираться и храниться на серверах в РФ**. Штрафы:
   1–6 млн ₽ за первое нарушение, 6–18 млн ₽ за повторное. Neon (США) — прямое нарушение.

**Поэтому:**

| Аудитория | Рекомендованный стек |
|---|---|
| 🇷🇺 РФ / СНГ | **Полностью российский** (Yandex Cloud, либо Timeweb/Selectel). 152-ФЗ + нет троттлинга. |
| 🌍 Международная | Vercel + R2 + managed Postgres (Supabase/DO). Проблема — **оплата из РФ**. |
| 🇷🇺+🌍 оба | Сложно: Cloudflare режется в РФ. Российский origin + отдельная международная раздача. |

> Приложение называется «WhosMeowing / cat arena» (англоязычное), но ты в РФ и склоняешься к
> Yandex. **Если целишь в российских пользователей — дальше читай «Российский стек».** Если
> аудитория международная и оплата зарубежных карт решена — «Международный стек».

## 1. Сервисы, которые нужны ТОЧНО (независимо от стека)

Каждому приложению этого типа нужны 6 «кирпичей». Вот они и за что отвечают:

| # | Кирпич | Зачем именно тебе | Платное? |
|---|---|---|---|
| 1 | **Compute (Node-сервер)** | Крутить Next.js. ⚠️ У тебя `sharp` (нативные бинарники) + Prisma → нужен **настоящий Node**, не edge/pure-serverless. | Да |
| 2 | **PostgreSQL** | Все данные: коты, голоса, рейтинги Glicko, юзеры, сессии. Уходишь с Neon. | Да |
| 3 | **Object Storage** | Хранить картинки котов (оригинал + WebP thumb/card). S3-совместимое → `@aws-sdk/client-s3` работает без переписывания. | Да (дёшево) |
| 4 | **CDN** | Раздавать картинки быстро. **Egress (исходящий трафик) — твой главный риск расходов** при вирусном всплеске. | Да (по трафику) |
| 5 | **Redis** | Сейчас: rate-limit голосов + одноразовый nonce токена пары. | Да (мало) |
| 6 | **Домен** | Брендовый HTTPS, SEO, email-домен. | Да (копейки) |

Плюс «вспомогательные» (стартуют с бесплатных тарифов): email (Resend/Sendersy),
мониторинг ошибок (Sentry), аналитика (PostHog/Метрика), uptime-мониторинг.

### Вариант A — Yandex Cloud (максимум managed, меньше DevOps)

| База | Yandex Managed PostgreSQL (1 хост) | ~1800–6000 ₽ | пример в их прайсе «от $20.38/мес» ≈ 1830 ₽ — **проверь калькулятором** |
| Storage | Yandex Object Storage | ~300–600 ₽ | картинки, S3-совместимо |
| CDN | Yandex CDN | 150 ГБ/ресурс free, дальше per-GB | раздача картинок, **не троттлится в РФ** |
| Redis | Yandex Managed Valkey (1 хост) | ~1500–2500 ₽ | rate-limit + nonce |
| **Итого** | | **~5000–9000 ₽/мес** | |

**Плюсы:** всё в РФ-юрисдикции (152-ФЗ ✓), оплата рублёвой картой без трения, самый зрелый
managed-Postgres среди РФ (бэкапы, PITR, failover), отличное покрытие CDN по РФ/СНГ.
**Минусы:** дороже бюджетных; managed-Postgres — крупнейшая статья.

> ⚠️ **Важно про пулер:** Yandex MDB использует **Odyssey** (свой пулер, не PgBouncer). Для
> Prisma в connection string нужно `?pgbouncer=true` (отключает prepared statements). Прямого
> порта 5432 «в обход пулера» нет — `DIRECT_URL` для миграций настраивается сложнее, чем у
> Neon/Supabase (отдельный route Odyssey в session-режиме или `connection_limit=1`).

### Вариант B — Timeweb Cloud (самый дешёвый серьёзный РФ)

| Компонент | ~Цена/мес ⚠️ | Заметка |
|---|---|---|
| VPS Cloud 2 vCPU / 4 GB | ~550–900 ₽ | проверь актуальный тариф (цены подняли в дек 2025) |
| Managed PostgreSQL | **от 230 ₽** | дешевле всех РФ; но managed-функции беднее Yandex |
| S3 storage 50 ГБ | ~200–400 ₽ | |
| CDN | ~200–500 ₽ | |
| Redis | **self-hosted на том же VPS** (free) | нет managed Valkey |
| **Итого** | **~1200–2500 ₽/мес** | |

**Плюсы:** в 3–4 раза дешевле Yandex, оплата рублём/СБП, 152-ФЗ ✓.
**Минусы:** managed-Postgres менее зрелый (нет HA на дешёвых тарифах), Redis крутишь сам,
CDN-сеть меньше Yandex.

### Вариант C — Selectel (золотая середина)

Дороже Timeweb, дешевле Yandex (~3000–6000 ₽/мес). **Важный плюс:** их Managed PostgreSQL
**имеет встроенный PgBouncer** (transaction-режим) — это лучше ложится на Prisma+Next.js, чем
Odyssey у Yandex. Есть «бюджетный» класс хранилища (0.79 ₽/ГБ) и CDN с 55+ PoP по РФ/СНГ.

### Что менять в коде под российский стек

- **Object Storage:** сменить `S3_ENDPOINT` (+bucket, credentials) на `storage.yandexcloud.net`
  (или Selectel/Timeweb). Весь `@aws-sdk/client-s3`-код остаётся как есть. ✅ минимальное изменение.
- **Redis:** ⚠️ **главная боль.** `@upstash/ratelimit` принимает **только** Upstash HTTP-клиент —
  он **не** работает с ioredis/Valkey ([upstash issue #115](https://github.com/upstash/ratelimit-js/issues/115)
  открыт). Чтобы уйти с Upstash, надо **переписать rate-limit** на `ioredis` + Lua / на
  `rate-limiter-flexible`. Это реальная работа, не «поменять адрес».
- **Email:** Resend → **Sendersy** (РФ-серверы, рублёвая оплата, похожий API; free 3000/мес,
  100/день; Pro 1600 ₽/50k). Ты рендеришь React Email в HTML на сервере и шлёшь HTML — должно
  лечь, но **проверь** перед миграцией.
- **Контейнер на Serverless:** ⚠️ если поедешь на Yandex Serverless Containers — нужен
  `output: 'standalone'` в `next.config.ts` (сейчас **не стоит**). На обычной VM не нужно.

### Домен и DNS для РФ

- **.ru** на reg.ru — **169 ₽ первый год** (продление дороже), рублёвая оплата. Хорош для РФ-SEO.
- ⚠️ **DNS:** даже **non-proxied** Cloudflare-неймсерверы рискованны в РФ (их IP попадали под
  блокировки). Для РФ-аудитории лучше делегировать DNS российскому провайдеру (Yandex DNS,
  reg.ru, Selectel), а не полагаться на Cloudflare.

## 4. Нужны ли очереди (Sidekiq-style)? — **Нет, пока не нужны**

Честный разбор по задачам приложения:

| Задача | Сейчас | Вердикт |
|---|---|---|
| **Glicko-2 рейтинги** | синхронно в транзакции с голосом | ✅ **обязан** оставаться синхронным. В очередь нельзя — рассинхрон голос/рейтинг. |
| **Обработка картинок (sharp)** | inline в POST /api/cats | ОК сейчас: браузер грузит в R2 напрямую (большой файл не идёт на сервер), sharp быстр (100–400 мс). В очередь — только при bulk-загрузке / частых таймаутах. |
| **AI-модерация (Workers AI)** | inline | Уже fail-safe: при ошибке/лимите → PENDING. Очередь не нужна. ⚠️ Free-лимит 10k Neurons/день ≈ **8300 проверок/день**; сверх — ошибки (PENDING-фолбэк это ловит). |
| **Email (Resend)** | синхронно в Auth.js | ✅ правильно: при ошибке отправки юзеру надо сказать сразу. |
| **Периодика (RD-decay, чистка)** | ❌ нет | **Единственный реальный пробел.** Нужен **cron**, не очередь. |

**Минимально правильно:** оставить всё inline + добавить **один cron**:
- **RD-decay** (раздувать rating deviation у неактивных котов, чтобы возвращались в пул) — важно
  для корректности Glicko-2. Раз в сутки.
- **Чистка R2-сирот** (originals без записи в БД) — раз в неделю.

Реализация: на VPS — системный crontab → защищённый `/api/cron/decay`. На Vercel — Vercel Cron
(на Hobby до 100 cron-задач, но минимум раз в сутки — этого хватает).

**Очередь (pg-boss) добавлять позже**, когда появится постоянный Node-процесс на VPS и реальные
таймауты. pg-boss лучше BullMQ для тебя: живёт в твоём Postgres, без отдельного Redis (а BullMQ +
Upstash имеет [баг с TLS](https://github.com/taskforcesh/bullmq/issues/3193)).

## 7. Порядок миграции (фазы)

1. **Домен + DNS** — купить, настроить (фундамент для email-домена и SEO).
2. **База** — поднять Postgres (managed РФ или international), `pg_dump` из Neon → restore,
   переключить `DATABASE_URL`/`DIRECT_URL`. **Сначала проверить пулер** (Odyssey `?pgbouncer=true`).
3. **Storage + CDN** — создать bucket у нового провайдера, перелить картинки, сменить `S3_ENDPOINT`,
   поставить CDN перед bucket.
4. **Compute** — задеплоить Next.js (Docker на VM, либо Vercel). sharp проверить на проде.
5. **Redis** — ⚠️ если уходишь с Upstash: переписать rate-limit на ioredis/`rate-limiter-flexible`.
6. **Email** — verified-домен + SPF/DKIM/DMARC (иначе magic-link в спам). РФ → Sendersy.
7. **Cron** — RD-decay + чистка сирот.
8. **Мониторинг** — Sentry DSN, uptime (Betterstack/Gatus), Метрика/PostHog.

## 8. Жёсткие ограничения (запомнить)

- 🔴 **sharp ⇒ настоящий Node.** Никакого pure-edge. (В репо уже настроено для Node-runtime.)
- 🔴 **Cloudflare/Hetzner/DO/OVH троттлятся в РФ** (с 9 июня 2025). Для РФ-аудитории — РФ-хостинг.
- 🔴 **152-ФЗ:** email+OAuth граждан РФ — на серверах в РФ (с 1 июля 2025), штраф до 18 млн ₽.
- 🔴 **`@upstash/ratelimit` не ioredis-совместим** — уход с Upstash = переписать rate-limit.
- 🟡 **Egress картинок — главный риск расходов.** На R2 = $0; на Yandex/Selectel ~1.3–1.7 ₽/ГБ
  сверх free-тира (10 ТБ/мес ≈ 16 000 ₽). Моделируй трафик заранее.
- 🟡 **Пулер обязателен** (Odyssey/PgBouncer) — иначе при всплеске упрёшься в `max_connections`.

> Все цены — ориентир на 2025–2026. ⚠️ перед оплатой сверяйся с калькулятором провайдера:
> Yandex/Selectel меняли цены в 2026, точные РФ-тарифы за CAPTCHA.
